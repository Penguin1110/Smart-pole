# Data Catalog

## Active Data

- `data/timeseries_data.json`  
  Main PM2.5 time series. Contains SP05 raw readings, SP05 hourly values, and EPA station values.

- `main/wind_pm25_data.js`  
  Frontend-ready combined dataset generated by `preprocess_wind_pm25.py`. Includes CWA wind and weather fields such as `WD`, `WS`, `RH`, `Temp`, `Td`, `StnPres`, `SeaPres`, `WSGust`, `WDGust`, `Precp`, `PrecpHour`, `SunShine`, `GloblRad`, `UVI`, and `Cloud`.

- `main/peak_data.js`  
  Frontend-ready peak-day summary used by the dashboard.

- `main/event_diagnosis_data.js`  
  Event-level diagnosis generated by `preprocess_wind_pm25.py`.

- `data/peak_hourly_clean.json`  
  Clean human-readable peak-day hourly structure. Use this instead of `data/peak_hourly.json` for manual inspection.

- `data/daily_avg.json`  
  Daily PM2.5 averages.

- `data/cwa_weather_inventory.json`  
  Coverage inventory for CWA weather fields by station.

## Legacy / Avoid For New Analysis

- `data/peak_hourly.json`  
  Legacy hourly peak file. It contains repeated null rows and should not be used as the canonical hourly peak source.

- `data/peak_hourly2.json`  
  Older clean source copied to `data/peak_hourly_clean.json`. Kept for traceability.

- `data/peak_days_all.json`  
  Useful for historical peak-day ranking, but `main/peak_data.js` is what current frontends load.

## Raw Data

- `CWA_Weather_Final/`  
  Raw CWA weather CSV folders.

- `CWA_Weather_Final/其他/`  
  Optional extra PM2.5 CSV inputs. `preprocess_wind_pm25.py` aggregates files here when they include `deviceId`, `PM2.5`, `time`, `lon`, and `lat`.

## Legacy Helpers

- `legacy/dataFetcher.js`
- `legacy/smartpole_static_data.js`
- `legacy/config.example.js`

These files are kept for reference from an older static/live API workflow. The current dashboard does not load them.

## Recommended Event Diagnosis Rules

- Treat `WS < calm_threshold` as calm; do not infer source direction from WD in those hours.
- Compare `PM2.5(t)` with `Wind(t-lag)` for lag values from `-6h` to `+6h`.
- Use `sin(WD)` and `cos(WD)` or wind vectors for correlation, not raw 0-360 degree values.
- Use bivariate polar plots and high-PM probability by wind direction and wind speed.
- Classify events as `transport`, `stagnation`, `regional`, or `local/mixed`.
