# Dashboard Map

## Entry Point

- `dashboard_index.html` - Root entry page. Links to the current main dashboard.
- `main/wind_pm25_lag_dashboard.html` - Main event diagnosis dashboard for PM2.5 and CWA wind/weather analysis.

## Main Frontend Structure

- `main/wind_pm25_lag_dashboard.html` - Page structure and control markup.
- `main/css/wind_pm25_lag_dashboard.css` - Dashboard layout, map, panel, marker, chart, and responsive styles.
- `main/js/wind_pm25_lag_dashboard.js` - UI state, map rendering, event diagnosis rendering, charts, lag logic, and polar plot.

## Generated Frontend Data

- `main/peak_data.js` - Peak event data consumed by the dashboard.
- `main/wind_pm25_data.js` - Combined CWA wind/weather and PM2.5 data generated by `preprocess_wind_pm25.py`.
- `main/event_diagnosis_data.js` - Event-level diagnosis generated by `preprocess_wind_pm25.py`.

## Source / Inspection Data

- `data/timeseries_data.json` - Main PM2.5 time series consumed by preprocessing.
- `data/peak_hourly_clean.json` - Preferred human-readable peak-hourly file for manual inspection.
- `data/cwa_weather_inventory.json` - CWA weather-field coverage inventory by station.

## Current Interpretation Rules

- Use lagged pairing: `PM2.5(t)` with `Wind(t - lag)`.
- Treat wind speed below the calm threshold as unreliable for directional source inference.
- Prefer wind direction plus wind speed polar views over direction-only summaries.
- Interpret peaks as transport, stagnation, regional, or local/mixed events rather than assuming every peak has a single immediate upwind source.
